# Milestone v1.0: MVP

**Status:** SHIPPED 2026-02-10
**Phases:** 1-4
**Total Plans:** 11

## Overview

This roadmap delivered a mobile-first PWA for real-time audio transcription with AI-powered summaries. Starting with the foundation (auth, recording, live transcription), adding file upload flexibility, building organization features (search, tags, filters), and finishing with AI intelligence (summaries, action items). Each phase delivered verifiable user value and unblocked the next.

## Milestone Summary

**Key Decisions:**
- Convex Auth with password provider (session persistence across browser restarts)
- Deepgram REST API for chunk transcription (simpler than WebSocket, reliable)
- Convex actions as API proxy (keep keys server-side)
- MediaRecorder MIME fallback chain (iOS Safari compatibility)
- Zustand for recording state management
- HTML5 Audio for playback (no third-party player)
- Junction table for tags (many-to-many)
- Lazy-load jsPDF via dynamic import()
- Web Share API first, anchor download fallback
- Claude haiku-4-5 for cost-effective AI summarization
- Prompt-based JSON output from Claude (no structured outputs)
- buildDeepgramUrl helper for dynamic URL construction from user settings

**Issues Resolved:**
- iOS Safari 44-byte audio blob bug (size > 44 check)
- Claude JSON response wrapped in markdown fences (strip before parse)
- MediaRecorder format compatibility across browsers (fallback chain)

**Issues Deferred:**
- Physical iOS device testing (simulators insufficient for full audio validation)
- Existing transcripts missing fullText (search only finds newly completed transcripts)

**Technical Debt Incurred:**
- None significant
